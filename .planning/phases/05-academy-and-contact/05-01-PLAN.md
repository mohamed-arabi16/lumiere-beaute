    ```typescript
    interface ButtonProps {
      variant: 'primary' | 'ghost';
      children: React.ReactNode;
      className?: string;
      onClick?: () => void;
      as?: 'button' | 'a';
      href?: string;
      type?: 'button' | 'submit' | 'reset';
      disabled?: boolean;
      target?: string;   // for external links opened in new tab
      rel?: string;      // rel="noopener noreferrer" for _blank security
    }
    ```
    Update the destructured props signature to include `target` and `rel`.
    Spread both props onto the `<Tag>` element — they are ignored by browsers on `<button>` and work correctly on `<a>`. The existing spread approach already passes `href`, so add `target` and `rel` in the same position:
    ```tsx
    return (
      <Tag
        href={href}
        target={target}
        rel={rel}
        type={Tag === 'button' ? type : undefined}
        onClick={onClick}
        disabled={Tag === 'button' ? disabled : undefined}
        className={[base, variants[variant], className].filter(Boolean).join(' ')}
      >
        {children}
      </Tag>
    );
    ```
    Do NOT change any styling, variant logic, or existing props. Only add the two new optional props.
